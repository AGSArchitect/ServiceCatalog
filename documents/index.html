Amazon Elastic Compute Cloud

Amazon EC2 User Guide: The Amazon EC2 User Guide is the definitive technical manual for anyone looking to master the Amazon Elastic Compute Cloud ecosystem. It provides a comprehensive roadmap for launching and managing virtual servers, known as instances, within the AWS cloud infrastructure. The document guides users through critical workflows, including selecting the right Amazon Machine Image (AMI) to serve as a software template and choosing between various instance types optimized for compute, memory, or storage-intensive workloads. Beyond basic deployment, the guide covers essential administrative tasks such as configuring security groups to act as virtual firewalls, managing persistent block storage via Amazon EBS, and setting up static IP endpoints. Whether you are a beginner following step-by-step tutorials to launch your first instance or an experienced architect implementing auto-scaling and high-availability strategies, this guide offers the technical depth needed to build secure and scalable solutions. It remains an indispensable resource for maintaining full control over your virtualized environment while ensuring operational efficiency.

Amazon EC2 Instance Types: The Amazon EC2 Instance Types document is an essential technical guide designed to help users navigate the extensive range of virtual hardware configurations available within the Amazon Elastic Compute Cloud ecosystem. It categorizes instances into specialized families—General Purpose, Compute Optimized, Memory Optimized, Storage Optimized, and Accelerated Computing—each tailored to specific workload demands. The guide provides detailed specifications for vCPUs, memory, network performance, and storage capacity, ensuring that developers and architects can balance performance with cost-efficiency. Beyond basic hardware stats, it explores advanced architectural features like the AWS Nitro System, burstable performance options, and specialized instances for high-performance computing or machine learning. By detailing naming conventions and regional availability, the document allows users to make informed decisions when scaling applications. Whether you are hosting a small web server or a massive data-processing cluster, this resource is the definitive reference for optimizing your cloud infrastructure's underlying performance.

AWS Nitro Enclaves User Guide: The AWS Nitro Enclaves User Guide is a specialized technical resource designed to help users create highly isolated compute environments, known as enclaves, within Amazon EC2 instances. This documentation focuses on the architecture and implementation of these hardened, virtual machines that possess no persistent storage, external networking, or interactive access—not even for root or administrator users. The guide provides step-by-step instructions for the complete enclave lifecycle, from launching an enclave-enabled parent instance to building an Enclave Image File (EIF) using the Nitro CLI. It also delves into critical security features such as cryptographic attestation, which allows applications to prove their identity to external services like AWS KMS before accessing sensitive data. By detailing how to manage CPU and memory allocation through the Nitro Hypervisor, the guide ensures that developers can process highly sensitive information—such as financial records or personally identifiable data—with a significantly reduced attack surface. Whether you are automating deployments with the AWS SDK or manually configuring local vsock communication, this document is essential for mastering confidential computing on AWS.

Amazon EC2 Developer Guide: The Amazon EC2 Developer Guide is a comprehensive technical document designed for software engineers and architects who are building and integrating applications with Amazon Elastic Compute Cloud. It serves as a deep-dive manual that focuses on the programming interfaces and specialized tools required to automate, manage, and scale virtualized infrastructure. The guide provides extensive code examples across multiple programming languages, demonstrating how to use AWS SDKs to perform essential operations such as launching instances, managing security groups, and configuring persistent storage. Beyond basic setup, it delves into complex developer-centric topics including API request signing, error handling, and the use of the AWS Command Line Interface (CLI). By centralizing best practices for security and resource management, the guide ensures developers can create robust, high-performance cloud applications while maintaining precise programmatic control over their compute environment. This resource is essential for any developer looking to leverage the full power of the EC2 API and its diverse capabilities.

Amazon EC2 API Reference: The Amazon EC2 API Reference document is the definitive technical manual for developers and systems architects requiring granular, programmatic control over their cloud infrastructure. It serves as an exhaustive catalog of every available action within the Amazon Elastic Compute Cloud, ranging from fundamental tasks like launching and terminating instances to advanced operations such as managing VPC peering, configuring network interfaces, and handling complex block storage volumes. Each API action is meticulously documented with its required and optional parameters, response structures, and potential error codes, ensuring that automated scripts and custom applications can interact reliably with AWS services. The reference also provides vital information on request signing, query authentication, and the specific data types used across the service, making it an essential companion to the AWS SDKs and Command Line Interface. By detailing the underlying mechanics of the EC2 service, this document empowers users to build sophisticated, self-healing, and highly automated cloud environments that can scale dynamically to meet the needs of any modern enterprise workload. For more information, visit the official Amazon EC2 API Reference.

Amazon RDS and Aurora

Amazon RDS User Guide The Amazon RDS User Guide is the definitive technical manual for developers and database administrators seeking to manage relational databases within the AWS cloud environment. This comprehensive document details how to efficiently set up, operate, and scale a relational database, supporting popular engines such as MySQL, PostgreSQL, MariaDB, Oracle, and Microsoft SQL Server. It provides a clear roadmap for automating critical yet repetitive administrative tasks, including hardware provisioning, software patching, and automated backups. The guide also delves into high-availability strategies like Multi-AZ deployments and read replicas to ensure data durability and performance. Security is a primary focus, with detailed instructions on configuring VPC isolation, IAM database authentication, and encryption at rest using AWS KMS. Whether you are a beginner using the "Easy Create" method or an expert managing global clusters, this resource provides the necessary insights to build a secure, cost-effective, and high-performance database infrastructure that scales dynamically with your application's needs.

The documentation also outlines the fundamental structure of the RDS query API, including how to authenticate requests and handle various data formats. By utilizing this reference, teams can automate complex database management workflows—such as scaling compute resources or orchestrating cross-region failovers—directly through the AWS SDKs or the Command Line Interface (CLI). This programmatic approach allows for greater operational agility and consistency, making the API Reference a cornerstone for anyone architecting high-performance, automated database solutions within the AWS cloud ecosystem.

Amazon RDS Data API Reference The Amazon RDS Data API Reference is an essential technical resource for developers looking to interact with Amazon Aurora databases using a web-services interface. Unlike traditional database connections that require persistent drivers and complex connection pooling, the Data API allows you to execute SQL statements via a secure HTTP endpoint. This reference document provides a complete catalog of API actions, such as executing statements, performing batch operations, and managing transactions, all without the overhead of managing long-lived connections. Each entry includes detailed descriptions of request parameters, response structures, and common error codes, enabling seamless integration with serverless applications like AWS Lambda. By detailing how to use the AWS SDKs to communicate with your Aurora clusters, the guide ensures that developers can focus on application logic while the Data API handles the underlying connection scaling and security authentication via AWS Secrets Manager. It is a critical manual for anyone architecting modern, scalable, and highly available cloud-native database solutions.

Elastic Load Balancing

User Guide for Application Load Balancers The User Guide for Application Load Balancers is a foundational technical manual for implementing layer 7 load balancing within the Amazon Web Services ecosystem. It provides comprehensive instructions on how to route incoming HTTP and HTTPS traffic to various targets, including Amazon EC2 instances, containers, and Lambda functions. The guide details advanced request-routing features, such as path-based and host-based routing, which are essential for modern microservices and containerized architectures. Additionally, it covers critical configuration components like listeners, target groups, and health checks to ensure high availability and application reliability. Security is also a major focus, with sections dedicated to SSL/TLS termination, web application firewall integration, and user authentication. By following this guide, developers and system administrators can optimize application performance, simplify traffic management, and build robust, scalable cloud infrastructures that respond dynamically to fluctuating user demands. This documentation is indispensable for anyone looking to provide a seamless and secure entry point for their web applications while maintaining granular control over traffic distribution and back-end health monitoring.

Networking

AWS Transit Gateway AWS Transit Gateway serves as a regional virtual router that streamlines cloud networking by interconnecting thousands of Amazon Virtual Private Clouds and on-premises networks through a centralized hub. The technical documentation, located at the AWS documentation site, provides an exhaustive overview of the service, detailing how it eliminates complex mesh peering and allows for easier scalability. Administrators can find critical information regarding resource attachments, route table management, and the implementation of Equal Cost Multipath routing to optimize data throughput. The guide explains how the gateway operates at Layer 3 to ensure traffic remains within the secure AWS global backbone while offering high availability. Furthermore, the documentation covers advanced topics such as peering gateways across regions, integrating Site-to-Site VPNs, and utilizing Direct Connect for hybrid connectivity. It serves as an essential reference for architects aiming to build a secure, efficient, and highly integrated multi-VPC infrastructure.

Storage

Amazon S3 User Guide The Amazon S3 User Guide is a comprehensive technical document designed to help users harness the full potential of Amazon Simple Storage Service for scalable, high-speed object storage. Located at the official AWS documentation site, this guide provides in-depth instructions for storing and retrieving any amount of data from anywhere on the web. It covers foundational concepts such as creating buckets, managing objects, and understanding the flat, non-hierarchical namespace. More advanced sections detail critical features like storage classes for cost optimization—ranging from S3 Standard to S3 Glacier—and lifecycle policies for automated data transitions. Security is a primary focus, with extensive documentation on Identity and Access Management, bucket policies, and server-side encryption to protect data at rest. Additionally, the guide explains modern capabilities like S3 Object Lambda, Multi-Region Access Points, and strong read-after-write consistency. Whether you are a developer integrating S3 via the AWS SDKs or an administrator managing large-scale data lakes, this manual serves as the definitive resource for maintaining a secure, durable, and highly available cloud storage infrastructure.
